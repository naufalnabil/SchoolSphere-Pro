# PHP Native School CMS Skill

## Description

Gunakan skill ini saat mengerjakan proyek SchoolSphere-Pro, yaitu website sekolah modular berbasis PHP Native Modular, Laragon, MySQL/phpMyAdmin, dan frontend-first.

## Core Rules

1. Gunakan PHP native modular.
2. Jangan gunakan React, Supabase, Laravel, atau stack lain tanpa izin.
3. Frontend public harus dikerjakan lebih dulu.
4. Jangan buat database sebelum UI public disetujui.
5. Jangan buat admin sebelum landing page public matang.
6. Jangan campur query database dan view.
7. Setiap section public harus menjadi komponen independen.
8. Jangan membuat dummy image.
9. Jangan membuat konten palsu production.
10. Selalu pecah tugas besar menjadi tahap kecil.

## Required Workflow

Sebelum mengubah file:
1. baca dokumen di folder `docs/`;
2. jelaskan rencana;
3. sebutkan file yang akan diubah;
4. tunggu konfirmasi jika perubahan besar.

Setelah mengubah file:
1. laporkan file dibuat/diubah;
2. jelaskan cara tes;
3. sebutkan yang tidak disentuh;
4. jangan lanjut tahap berikutnya tanpa perintah.

## Frontend Principles

- premium school website;
- invisible grid;
- adaptive spacing;
- smart background;
- wide card rule;
- no dummy image;
- responsive;
- smooth animation;
- professional footer;
- sticky navbar;
- strong hero.

## PHP Rules

- gunakan helper path;
- gunakan component include;
- gunakan fixture development sebelum database;
- gunakan PDO saat database dibuat;
- gunakan prepared statement;
- escape output dengan helper `e()`.

## Data Safety

Jangan tampilkan data sensitif:
- data pendaftar;
- data siswa kelulusan;
- nilai;
- NIK;
- password;
- admin logs.

## When Unsure

Jika instruksi ambigu, tanyakan sebelum melakukan perubahan besar.
